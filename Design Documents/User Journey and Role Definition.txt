
Journey 1: New Player — First Booking

Land on Home → Sign Up (email + password) → Verify email (Supabase)
→ Complete Profile (name, phone, accept waiver)
→ Browse Availability (select facility, date, duration)
→ Select time slot → Select court (or auto-assign)
→ View pricing summary + policy
→ Click "Book & Pay" → Stripe Checkout (hosted)
→ Enter card → Pay → Redirect to /payment-success?session_id=xxx
→ App verifies Stripe session server-side → Booking confirmed
→ Confirmation page shown → Booking visible in "My Bookings"


Journey 2: Cancel Booking

Log In → My Bookings → Select booking
→ View cancellation policy (based on time remaining)
→ Confirm cancellation → Stripe refund initiated (if eligible)
→ Booking status → "cancelled" → Email placeholder triggered


Journey 3: Facility Admin — Configure New Court

Log In (admin role) → Admin Dashboard → Facility Config
→ Select facility → Courts tab → Add Court
→ Enter name, sport type, status → Save
→ Audit log entry created automatically

Journey 4: Facility Admin — Block Dates

Admin Dashboard → Facility Config → Closures tab
→ Add closure (date + optional time range + reason)
→ Save → All slots in that range removed from availability
→ Existing confirmed bookings flagged for admin review


Journey 5: Super Admin — Monitor Revenue

Log In → Admin Dashboard → All Facilities view
→ Revenue metrics by facility / date range
→ Utilization heatmap by court/time
→ Export-ready data table

Super Admin
